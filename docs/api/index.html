jeli Module

# jeli(name, options)

  • Arguments

    • {string} name (required)
    • {object} options

  • Return

    {object} jeli instance

  • Usage

    Create a new "jeli module Instance", options argument is required when creating a module for the first time. The options should contain the module configuration

    Modules created can be used to bootstrap application, or used as required modules.

# Module.config(configuration)

  • Arguments

    • {Function} configuration

  • Dependency Injection: Yes

  • Usage

    Set configuration for a module using the jConfig api, table below shows the injectable dependencies for jConfig:

    Service Providers Value
    No yes Yes

Once application bootstrap, all configuration are first initialized.

# Module.provider(providerName, providerFn)

  • Arguments

    • {string} providerName
    • {Function} providerFn

  • Dependency Injection: No

  • Usage

    The provider is useful for setting and getting module configuration. $get api must be defined in the provider instance. Below table shows injectable dependencies for $get api

    Factory Service Providers Value
    Yes Yes Yes Yes
# Module.service(name, [definition])

  • Arguments

    • {string} name
    • {Function} definition

  • Dependency Injection: Yes

  • Usage

    As jEli apis are singleton, service api can be used to get new Instance of a Function.
# Module.filter(name, [definition])

  • Arguments

    • {string} name
    • {function} definition

  • Dependency Injection: Yes

  • Usage

    Register a global filter.
# Module.value(name, [definition])

  • Arguments

    • {string} name
    • {Function | Object} definition

  • Dependency Injection: No

  • Usage

    Register a global value.
# Module.element([definition])

  • Arguments

    • {object} [definition]

  • Dependency Injection: Yes

  • Usage

    Register a global component that can be used to extend html.
# Module.directive(name, [definition])

  • Arguments

    • {string} name
    • {function} definition

  • Dependency Injection: Yes

  • Usage

    Register a global directive.
# Module.jInit([definition])

  • Arguments

    • {function} definition

  • Dependency Injection: Yes

  • Usage

    Register a module initializer.

jEli ViewModel Api

# model.$new(isolate)

  • Arguments:

    • {boolean} isolate

  • Return: {object} Model Instance
# model.$evaluate(expr, context)

  • Arguments:

    • {string} expr
    • {object} context (optional)

  • Usage:

    Evaluate a valid binding expression on the current viewModel.
# model.$apply(callback)

  • Arguments:

    • {function} callback (optional)

  • Type: Function
# model.$watch(expr, callback)

  • Arguments:

    • {string | function} expr
    • {function} callback (optional)

  • Return: unwatch

  • Usage:

    Watch an expression or a computed function on the viewModel for changes. The callback gets called with the new value and the old value. The expression can be a single keypath or any valid binding expressions
# model.$consume()

  • Arguments: None

  • Type: {Function}

  • Details:

    Triggers every watch listner on a viewModel
# model.$destroy()

  • Arguments: None

  • Type: {Function}

  • Details:

    When called, all listeners and bindings to the viewModel are destroy and detached and also the model is destroyed and ready for GC.

jEli ViewModel Events

# model.$on(eventName, callback)

  • Arguments:

    • {string} eventName
    • {function} callback

  • Details:

    Listen for a custom event on the current vm. Events can be triggered by vm.$emit, vm.$dispatch or vm.$broadcast. The callback will receive all the additional arguments passed into these event-triggering methods.
# model.$broadcast(eventName, [...args])

  • Arguments:

    • {string} eventName
    • [...args]

  • Usage:

    Broadcast an event that propagates downward to all descendants of the current vm. Since the descendants expand into multiple sub-trees, the event propagation will follow many different “paths”. The propagation for each path will stop when a listener callback is fired along that path, unless the callback returns true.
