Introduction

What is jeliJS?

jeli (pronounced / J-ELI) is an Amazing and Progressive Javascript Framework for building User Interface without sacrificing power. As the name, jeli is just JavaScript written in modern and future approach.

jeli is very easy to pick up and integrate with other libraries or existing projects. On the other hand, jeliJS is also perfectly capable of powering sophisticated Single-Page Applications (SPAs). However, jeli can be used both on client and server side.

Rendering Our App

Let's back to our index.html file, did you notice the ${pageHeading} syntax? Well, inside of a view, you have access to all properties and methods of the View-Model instance associated with the view and you can access them inside the content of of any element or attribute by using jeli template string syntax as shown above. The above syntax creates a one-way data-binding to the associated property. By "one-way" we mean that the dataflow is unidirectional. The view only updates when the property binded to it changes. There is no "reverse" flow from the view back to the view-model.

We call a View's associated class its View-Model because it's a model for, or a model of the View. Most jeli development leverages the Model - View - View-Model pattern. However, jeli is flexible enough to enable also using patterns like Supervising Controller (as our todo App), Passive View and Model - View - Controller if desired.

We colect information from the user, when they tap enter in the input text box. How is this done? In the input element appended :model attribute, by doing so, we tell jeli that we want to bind "todoDescription" property in our View-Model. You can bind "Model" property to any HTML Element by appending :model .

Here, we have another convention that is important to point out. When we use :model jeli works on your behalf to pick the most sensible "binding mode" based on the element and attribute you are binding. For example, since this is an input and you are binding its value then :model will cause jeli to set up a two-way binding. This means that any time the view-model's todoDescription changes, it will be updated in the input's value property but also any time the input's value property changes, the view-model will have its todoDescription updated as well.

There's something else of note in this markup though. We can not only bind properties, but you can attach to events. Take a look at the form element itself. In jeli, you can attach any DOM event to an ELEMENT by using the symbol "@" followed by the event name. This means that when that event fires, it will trigger the associated expression to be evaluated. In this case, the submit event causes the addTodo() method to be invoked. Because we've used the submit event, it means that the todo will be added either by pressing the submit button or by pressing enter while inside the input

For more on event binding, please read on "Events" in jeli API section
